# AI Reflection

В процессе разработки проекта активно использовались инструменты искусственного интеллекта (в частности ChatGPT) как вспомогательный инструмент.

## Как использовался AI

AI применялся для:

- генерации структуры проекта (frontend + backend)
- помощи в написании backend на Express
- настройки Prisma и работы с базой данных
- реализации загрузки файлов через multer
- исправления ошибок (например:
  - ошибки JSON
  - проблемы с Prisma
  - ошибки Docker)
- написания React-компонентов
- улучшения UI и CSS
- настройки Docker и docker-compose
- структурирования проекта

## Что делалось самостоятельно

Несмотря на использование AI, значительная часть работы выполнялась самостоятельно:

- понимание архитектуры проекта
- интеграция frontend и backend
- исправление ошибок, которые AI не учитывал
- адаптация кода под реальные условия (например:
  - работа с файлами
  - настройка путей
  - исправление багов UI)
- принятие решений по архитектуре:
  - переход от base64 к файловому хранению
  - организация структуры CSS
- тестирование и отладка проекта

## Проблемы и их решение

Во время разработки возникали сложности:

- слишком большой размер изображений при хранении в базе
  → решено переходом на хранение файлов

- ошибки при работе с JSON и API
  → добавлена обработка текстового ответа

- проблемы со стилями
  → рефакторинг CSS и разделение по файлам

- сложности с Docker
  → исправлены Dockerfile и docker-compose

## Выводы

Использование AI значительно ускорило разработку, но:

- требовалась проверка и адаптация кода
- важно было понимать, как работает каждая часть системы

AI выступал как помощник, но не заменял понимание разработки.

В результате был создан полноценный fullstack проект с:

- frontend
- backend
- базой данных
- загрузкой файлов
- Docker-конфигурацией
