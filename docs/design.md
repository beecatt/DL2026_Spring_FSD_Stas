# Design

## 1. Общее описание

QR Studio — веб-приложение для генерации QR-кодов с возможностью кастомизации, добавления изображения в центр и сохранения истории.

Приложение реализовано по архитектуре client-server и состоит из frontend (React) и backend (Node.js + Express).

---

## 2. User Stories

### Основные сценарии пользователя

1. Как пользователь, я хочу ввести текст или ссылку, чтобы сгенерировать QR-код.

2. Как пользователь, я хочу изменять:
   - размер QR-кода
   - цвет QR-кода
   - цвет фона  
   чтобы настроить внешний вид.

3. Как пользователь, я хочу добавить изображение в центр QR-кода, чтобы сделать его брендированным.

4. Как пользователь, я хочу видеть QR-код в реальном времени, чтобы сразу понимать результат.

5. Как пользователь, я хочу скачать QR-код в формате PNG.

6. Как пользователь, я хочу сохранить QR-код, чтобы использовать его позже.

7. Как пользователь, я хочу видеть историю созданных QR-кодов.

8. Как пользователь, я хочу повторно использовать настройки из истории.

9. Как пользователь, я хочу удалить QR-код из истории.

10. Как пользователь, я хочу, чтобы приложение проверяло корректность URL.

11. Как пользователь, я хочу, чтобы мои настройки формы сохранялись при перезагрузке страницы.

---

## 3. Архитектура

Приложение разделено на две части:

### Frontend
- React (Vite)
- Управление состоянием через useState
- Работа с API через fetch
- sessionStorage для хранения состояния формы

### Backend
- Node.js + Express
- Prisma ORM
- SQLite база данных
- Multer для загрузки файлов
- Хранение изображений на сервере (uploads)

---

## 4. API

### Base URL


http://localhost:5000/api


---

### Получение всех QR-кодов


GET /qr


Ответ:


[
{
"id": "string",
"content": "string",
"size": number,
"foreground": "string",
"background": "string",
"centerImage": "string | null",
"createdAt": "date"
}
]


---

### Создание QR-кода


POST /qr


Тип: multipart/form-data

Параметры:

- content (string)
- size (number)
- foreground (string)
- background (string)
- centerImage (file, optional)

Ответ:


{
"id": "string",
...
}


---

### Удаление QR-кода


DELETE /qr/:id


Ответ:


{
"message": "QR code deleted successfully"
}


---

### Health check


GET /health


---

## 5. Модель данных

Используется SQLite база данных.

### Таблица: QRCode

Поля:

- id (string, primary key)
- content (string)
- size (number)
- foreground (string)
- background (string)
- centerImage (string, nullable)
- createdAt (datetime)

---

## 6. Работа с файлами

- изображения загружаются через multer
- сохраняются в папку `backend/uploads`
- в базе хранится путь к файлу
- при удалении QR-кода файл удаляется с сервера

---

## 7. Валидация

На стороне frontend реализована проверка:

- пустого значения
- корректности URL
- базовой структуры домена

---

## 8. Хранение данных

| Тип данных        | Где хранится         |
|------------------|----------------------|
| QR-коды          | SQLite (backend)     |
| Изображения      | файловая система     |
| Черновик формы   | sessionStorage       |

---

## 9. UI/UX решения

- мгновенный preview QR-кода
- единый стиль кнопок
- кастомный file input
- адаптивная верстка
- понятная структура формы

---

## 10. Используемые технологии

Frontend:
- React
- Vite

Backend:
- Node.js
- Express
- Prisma
- SQLite
- Multer

Дополнительно:
- Docker
- sessionStorage

---

## 11. Возможные улучшения

- добавление SVG экспорта
- добавление авторизации пользователей
- хранение изображений в облаке
- защита от дубликатов на уровне базы данных
- улучшение UX истории (мини-превью QR)

---

## 12. Вывод

Приложение реализует полный цикл работы с QR-кодами:

- генерация
- кастомизация
- сохранение
- повторное использование
- удаление

Решение построено с использованием современных технологий и демонстрирует навыки # Design
